## 0. 全局外壳（所有页面共用）

- `顶栏`：Logo/产品名 · 当前页标识 · 右侧 Profile 入口（头像）。
- `主区`：各页内容。
- `底栏`：版本/说明（demo 阶段够用）。
- 登录态：未登录只可见 登录/注册；已登录但**未绑定搭档**强制进 §0.5；绑定后才进下列页面。

## 0.5 绑定固定搭档（注册后一次性）　`/onboarding/partner`（待加路由）

注册后的强制一步，建立长期搭档关系；之后所有对局默认就是和 TA。

- `说明`：一句话讲清"CoStudy 是和一个固定搭档一起学"。
- `绑定方式`：生成我的邀请码/链接发给对方；或输入对方邀请码。
- `等待确认`：对方接受后绑定成功（双方互确认）。
- `→` 绑定成功 → `/`。
- 状态：未发起 / 等待对方确认 / 绑定成功 / 已被绑定（直接进首页）。
- 备注：解绑/换搭档为后续规划，先不在主流程出现。

## 1. 首页 / 我的资料　`/`

整个产品的中枢，回答"我有什么、要做什么"。

- `搭档条`：固定搭档头像 + 名字 + 当前「默契值」（始终在场，强调"你俩"）。未绑定时显示去绑定引导（→ `/onboarding/partner`）。
- `资料合集`（主区块）：学习包卡片列表。每卡：标题、来源图标（视频/文档/链接…）、处理状态、分段数、进度（已学段 / 已通关）。卡片 → 资料详情 `/material`。
- `主行动`：醒目的「导入资料」→ `/upload`；「发起对局」→ `/room/create`。
- `继续学习`：最近未完成的对局/资料段，一键回到现场。
- `统计概览`：轻量数字——已学资料数、累计通关、与固定搭档的默契值/平均评分。详情 → Profile。
- `Profile 入口`：顶栏头像 → `/profile`（见 §11，待加路由）。
- 状态：未绑定搭档（先引导绑定）/ 空态（无资料，突出「导入资料」）。

---

## 2. 导入资料　`/upload`

把文件**或链接**变成一个可学习/可出题的"学习包"。

- `学习包信息`：标题、可选描述。
- `两种来源（Tab/分区）`：
  - `上传文件`：拖拽/选择，支持 视频/音频/PDF/Word/PPT/文本/MD/图片；显示格式与大小限制。
  - `粘贴链接`：YouTube / Bilibili 视频，或网页文章/教程 URL；自动识别来源类型。
- `已选清单`：每行 文件名/链接、来源类型图标、大小（文件）、移除。
- `提交`：创建学习包 → 进入 `/processing`。
- 状态：未选任何来源时提交禁用；超限/不支持/链接无法抓取给出明确提示。

---

## 3. 资料处理中　`/processing`

解析 / 转录 / 出题进行中，给确定感，别让人干等。

- `总进度`：整体阶段（抓取/解析 → 分段 → 转录 → 出题 → 就绪）。
- `逐项状态`：每个文件/链接当前阶段 + 进度（长视频会久，单列出来）。
- `分段预览`：分段完成后可先看切了多少段（让用户对"能学多少"有预期）。
- `就绪后`：自动/手动 → `/material`（或直接「发起对局」→ `/room/create`）。
- 状态：处理中（轮询/实时刷新）、部分就绪（文本好了视频还在）、失败（错误原因 + 重试 / 链接抓取失败提示改上传文件）。

---

## 4. 资料详情 / 学习视图　`/material`

视频音频重点页：转录与 AI 总结并列，可互跳。也是建房前的"预习"。

- `资料头`：标题、来源类型、时长/页数、状态。
- `分段目录`：长资料按段（章节）列出，每段标题 + 进度（已学/已通关）；点段 → 定位该段内容，或直接据该段发起对局。
- `主区（左右两栏，不互相替代）`：
  - 左 `逐字转录`：带时间戳的文本流，点一句 → 定位原片/原文（事实层、可查证）。
  - 右 `AI 总结`：一句话概括 / 大纲 / 要点清单 / 术语表；每个要点可点 → 回链左侧对应位置（理解层）。
- `原片预览`（视频/音频时）：播放区，受转录/总结/分段点击驱动定位。
- `行动`：「发起对局」→ `/room/create`（可带上当前选中的段）。
- 状态：非视频资料（纯文本/PDF）隐藏播放区，左栏即原文；短资料可能只有 1 段（隐藏目录）。

> **demo 当前实现**（合并了 §5/§6 的"发起对局"+"等待就绪"）：
> - 资料按 **chunk（章）** 分组列 **pieces（节）**，每节状态 `locked｜available｜done｜skipped`。
> - 每节右侧两类操作：「**开始**」→ 选玩法 → 后端建对局 `status=invited` → 自动跳到 `/play/:sid`；「**提议跳过**」→ 写 `skip_requests`，搭档下次轮询时看到。
> - 搭档发起的待办在该节内联出现：「{名} 邀请你玩…」→ 接受/拒绝；「{名} 提议跳过…」→ 同意/不同意。同意跳过 → 此节 `state=skipped` + 解锁下一节。
> - 资料头部右上角：「**删除此资料**」（confirm 后级联清掉所有 chunk/piece/题目/对局/进度）。
> - 轮询 ~3s 自动刷新；前端无独立"处理中页"——导入即同步建表，邀请发起时才懒解析+出题。

## 5. 发起对局　`/room/create`

把"我有资料"变成"我们要一起闯关"。**搭档是固定的，无邀请步骤。**

- `选资料 + 选段`：选学习包，再选学哪一段（默认带入来源；长资料必选段）。
- `选玩法`：不对称选择题（信息不对称协作）/ 共答选择题（各自作答 + 分差评分）。
- `对局设置`：关卡数、限时。
- `通知搭档`：固定搭档自动收到对局邀约（无需输码/发链接）。
- `→` 创建后进入 `/room/lobby`。
- 状态：搭档在线（可直接发起）/ 离线（留邀约待 TA 上线）。

---

## 6. 房间 — 等待就绪　`/room/lobby`

开局前最后一步，双方确认。

- `对局信息`：资料名、所选段、玩法、关卡数、限时。
- `双人状态`：固定搭档两人在场 + 各自「已准备」状态。
- `玩法说明`：按所选玩法简述规则（不对称：讲解者/选择者会轮换；共答：各自作答、分差评分）。
- `开始`：双方都就绪才可开始 →
  - 不对称选择题 → `/play/narrator` 或 `/play/selector`（按本人角色）
  - 共答选择题 → `/play/co-choice`
- 状态：等待对方在场 / 等待对方准备 / 双方就绪。

---

## 7. 闯关中 — 讲解者视角　`/play/narrator`

核心体验之一。本人只看到题干，必须讲给对方。

- `关卡条`：第 X / 共 N 关、限时倒计时、我的角色=讲解者。
- `题干区`（仅本视角可见）：题目正文。强调"只有你能看到这部分"。
- `对方状态`：实时提示"搭档正在选择…"。
- `沟通辅助`：讲解输入/语音状态位（demo 占位即可）。
- `→` 双方完成本关 → `/play/reveal`。
- 状态：等待对方就位 / 进行中 / 已提交等揭晓。

---

## 8. 闯关中 — 选择者视角　`/play/selector`

与 §7 对称。本人只看到四个选项，看不到题干。

- `关卡条`：同 §7，我的角色=选择者。
- `选项区`（仅本视角可见）：四个选项，听完讲解后选择。
- `对方状态`：实时提示"搭档正在讲解…"。
- `提交`：选定后提交。
- `→` 揭晓 → `/play/reveal`。
- 状态：等待讲解 / 可选择 / 已提交等揭晓。

---

## 8.5 闯关中 — 共答选择题　`/play/co-choice`（待加路由）

玩法二。双方都看到完整题目，各自独立作答；靠分差评分驱动共同进步。

- `关卡条`：第 X / 共 N 关、限时、玩法=共答。
- `题目区`：完整题干 + 四个选项（双方都可见）。
- `我的作答`：选择并提交（对方答案在揭晓前不可见）。
- `对方状态`：实时提示"搭档已提交 / 思考中…"，不泄露其选项。
- `→` 双方提交/超时 → `/play/reveal`。
- 状态：作答中 / 已提交等对方 / 双方完成等揭晓。

---

## 9. 本关揭晓　`/play/reveal`

把一关收束成"共同结果"，并完成"边玩边学"。

- `结果`：对/错，措辞框定为"你们…"，不点名谁选错。
- `回链原文`：本题出自原文哪段 / 视频哪个时间点，可点开看原片片段。
- `失败处理`：鼓励语 + 可选 AI 提示（给思路不给答案）+ 重试。
- `推进`：进入下一关（角色轮换提示）或 → `/result`（最后一关）。
- 状态：通过 / 失败可重试 / 全部结束。

---

## 10. 结算页　`/result`

成就感与沉淀所在，决定"想不想再来"。

- `分差评分`（共答类玩法的核心区块）：两人各自得分、分差、最终等级（S/A/B/C）。
  - 显式解释规则：等级 = 总分表现 × 分差惩罚；**分差大即使有人很高也压低等级**。
  - 文案点出"一起进步才上分"，弱化"谁更强"。
  - 不对称选择题不显示此区块，只显示过关情况。
- `共同战果`：通关进度、combo、用时、攻克知识点数。
- `默契值`：这对固定搭档的关系数值，本局涨幅。
- `徽章`：本局解锁（首杀/无提示/速通/满 combo/小分差…）。
- `共同学习报告`：攻克 vs 易错知识点，每条可回链原文/原片。
- `分享`：成绩卡片。
- `复访钩子`：还剩 N 个易错点，「再战」/「回首页」。
- `→` 再战回 `/room/create`，或回 `/`。

---

## 11. Profile（建议新增路由 `/profile`）

首页统计点进来的个人/搭档中心。

- `个人信息`：头像、昵称、账号。
- `固定搭档`：搭档信息、默契值、共学资料数、最佳 combo、历史平均评分。
- `成就墙`：累计解锁徽章的聚合展示。**后续规划**——本期先占位（数据已由对局产出，届时只做聚合视图）。
- `学习足迹`：学过的资料/段、历史报告入口。
- `设置`：基础偏好（demo 可占位）。
