# CoStudy 设计向规划（临时·精简）

> 给设计稿用。只讲**产品功能**和**体验**，不含数据库/接口/部署细节（那些在 `PLAN.md`）。
>
> 一句话：两个**固定搭档**上传/导入一份资料，AI 出题，靠多种协作玩法一起闯关，按"分差更重要"的规则评分，沉淀成共同学习成果。

---

## 1. 产品是什么

- **谁用**：一对**固定搭档**（情侣、室友、固定学伴）。注册后即绑定一个长期搭档，之后所有学习都和 TA 进行——不是每次临时邀请，也不做陌生人匹配。
- **核心循环**：导入资料 → 自动分段 → AI 出题 → 双人闯关（多玩法）→ 按"分差更重要"评分 → 看共同战果 → 想再战。
- **差异点**：
  - 关系是**持久的**——成长（默契值、成就）属于这对固定搭档，越用越舍不得换。
  - **机制上强制互相依赖 / 强制共同进步**：要么信息不对称（必须靠对方），要么各自计分但**分差大就拿不到好成绩**——一个人 carry 没用。

## 2. 主要流程（按页面/状态走，给设计稿用）

```
[注册] → [绑定固定搭档]（邀请码互认，一次性）
            │
            ▼
[首页/我的资料] → [导入资料：上传文件 / 粘贴链接 + 备注]
                       │
                       ▼
            [处理中…：抓首页 → 章/节粗分（chunk + piece）]
                       │
                       ▼
   [资料详情：按"章"分组列节；每节有"开始 / 提议跳过"]
                       │
                ┌──────┴──────────────────────────┐
                ▼                                  ▼
       [发起邀请：选玩法]                  [提议跳过此节]
                │                                  │
                ▼                                  ▼
       [搭档接受 → playing]              [搭档同意 → 标 skipped + 解锁下一节]
                │
                ▼
  [闯关：不对称=一题一揭，左 7/10 嵌原网页，右 3/10 题；
        共答=各自答全部题]
                │
                ▼
        [结算：分差评分 / 合作通过]
```

demo 当前是**轮询**（前端 1.5–3s 拉一次）；后期换 WebSocket。

### 关键页面要点

**绑定固定搭档（注册后一次性）**
- 注册后第一步：用邀请码/链接和搭档互相确认，绑定成长期搭档。
- 绑定后才进首页；之后所有对局默认就是和 TA，无需每次邀请。

**导入资料（上传文件 / 粘贴链接）**
- 文件：视频、音频、PDF、Word/PPT、文本、Markdown、图片。
- 链接：YouTube / Bilibili 视频，或网页文章/教程 URL。
- 一份资料 = 一个"学习包"，可含多个文件/链接。
- 处理时自动**分段**（长视频/长文切成"一局能学完的段"，如按章节）。
- 处理中要有清晰进度（解析 / 分段 / 转录 / 出题），不能让人干等。

**资料详情 / 学习视图（视频音频重点）**
- **两块并列，不能互相替代**：
  - **逐字转录**：带时间戳，点一句可跳到对应视频/音频位置（事实层、可查证）。
  - **AI 总结**：一句话概括 + 大纲 + 要点清单 + 术语表（理解层）。总结里每个要点可点击 → **跳回原片对应时间点**。
- 设计上要让"快速看懂（总结）"和"回去查证（转录/原片）"无缝来回。
- **分段目录**：长资料按段（章节）列出，可单独学某段、看某段进度。

**发起对局（邀请 / 接受，不需"双人 ready"）**
- 资料详情页每个节（piece）右侧都有「开始」+「提议跳过」。
- 点「开始」→ 选玩法（不对称 / 共答）→ 服务端建一局 `status=invited`，搭档在 material 页或 /play 页看到「{名} 邀请你玩…」。
- 搭档**接受**才进入 playing；发起方可撤回，被邀请方可拒绝（同一 endpoint `/cancel`）。
- **跳过投票**：双方都觉得这一节不用做 → 一方点「提议跳过」→ 对方看到「搭档提议跳过此节」→ 同意/不同意。同意后此节 `state=skipped`，自动解锁下一节，不出题、不烧 AI。

**闯关中（核心体验，重点设计）**

布局统一：**左 7/10 嵌入原网页（iframe `source_url`），右 3/10 题目区**。X-Frame-Options 禁嵌的站点提供"在新标签打开"兜底。
- 玩法一·**不对称选择题**（一题一揭，demo 已实现）：
  - 讲解者（A）：左侧看原网页，右侧只看题干。
  - 选择者（B）：左侧"你看不到本节内容"（占位），右侧只看四个选项。
  - B 选完 → 双方同看揭晓（正确答案 + B 选了什么）→ 任一方点「下一题」→ 进入下一题。
  - 最后一题揭晓后点「结算」自动 finalize；当前 demo 角色一局内固定（每题轮换是下一步）。
- 玩法二·**共答选择题**（各自作答）：
  - 双方都看到原网页 + 完整题目，各自独立作答。
  - 不靠讲解，靠 §3 的"分差评分"驱动——想拿好成绩必须两人都答好且接近。
- 共通：屏幕清楚体现"当前玩法 / 我能看到什么 / 对方在做什么"；限时、第几题/共几题要明显。

**本关揭晓**
- 显示对错，但**框定为共同结果**："你们配合答对了！"，不点名谁选错。
- 立即展示这题出自原文哪一段 / 视频哪个时间点（可点开看原片片段）——边玩边学的闭环。
- 失败不羞辱：鼓励语 + 可选 AI 提示（给思路，不直接给答案），可重试。

**结算页（成就感所在）**
- **分差评分（重点）**：显示两人各自得分、分差、最终等级（S/A/B/C）。要明确传达规则——
  - 等级由"总分表现 × 分差惩罚"决定；**分差越大、等级压得越狠**。
  - 即使一人很高，只要分差大，也拿不到好等级；文案点出"再带带 TA / 一起进步才上分"。
  - （不对称选择题是合作单一结果，不走分差，仅显示过关情况。）
- 共享数据：通关进度、连击 combo、用时、本局攻克的知识点数。
- **默契值**：属于"你俩这对固定搭档"的成长数值（不是个人经验），随每次合作上涨。
- 解锁的徽章：首次通关 / 无提示通关 / 速通 / 满 combo / 小分差通关 / 一起学完 N 份资料。
- **共同学习报告**：这局你俩攻克了哪些知识点、哪些是易错点，每条可回链原文/原片。
- 可分享的成绩卡片；复访钩子："你俩还剩 2 个易错点，要不要重战？"
- （**成就墙**——历史徽章聚合展示——为后续规划，先在 Profile 占位。）

## 3. 体验原则（设计时贯穿）

1. **一切围绕"我们"**：固定搭档，成就/进度/数值都属于这对关系，不做个人排名/互相比较。
2. **共同进步优先**：评分里分差权重最高——一个人 carry 拿不到好成绩，引导"带着对方一起变好"。
3. **依赖即乐趣**：信息不对称是核心爽点，UI 要强化"只有我能看到这部分，必须靠对方"。
4. **玩完要有沉淀**：每局产出真实可用的学习报告，玩 ≠ 浪费时间。
5. **失败温和**：合作游戏里指责会破坏关系，错误一律框定为"一起再来一次"。
6. **AI 不抢戏**：AI 负责出题/总结/给思路，节奏和成就属于两个人。
7. **快慢两条路打通**：总结看懂 ↔ 原片查证，随时无缝切换。

---

## 4. 玩法

**已规划（首发设计需覆盖）**

- **不对称选择题**：信息不对称协作，合作单一结果（过/不过），不走分差评分。
- **共答选择题**：双方各自作答完整题目，走分差评分（分差大拿不到好成绩）。

**后续方向（先知道，暂不设计）**：接龙填空、限时抢答、一方限时讲解另一方答题、协作找茬（AI 给含错笔记）。

设计首发时保证"玩法 / 角色 / 可见信息 / 是否走分差评分"这套框架可复用，新增玩法不动其余系统。

---

## 5. 设计稿建议优先覆盖的画面

1. 闯关中——讲解者视角 & 选择者视角（两张，体现信息不对称）
2. 闯关中——共答选择题（双方各自作答）
3. 结算页（**分差评分**：双方得分、分差、等级与规则解释 + 默契值/徽章/报告）
4. 本关揭晓（含原文/原片回链）
5. 视频学习视图（转录 + AI 总结 + 分段目录 + 互跳）
6. 发起对局（选段 + 选玩法，固定搭档无邀请）
7. 绑定固定搭档（注册后一次性）
8. 导入资料（上传文件 / 粘贴链接）+ 处理中进度

其余（首页、资料列表、登录）按常规来即可。
